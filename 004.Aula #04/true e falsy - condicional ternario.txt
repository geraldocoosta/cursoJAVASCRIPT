Escopo de váriaveis:
no js existe o escopo global e escopo local
declarar uma váriavel na parte mais alta do código, fora de função, ela esta no escopo global
se declarar uma váriavel dentro de uma function, é criada uma função de escopo local
escopo global no js -> problema em algumas vezes
function -> cria escopo local
Se eu criar uma variavel dentro da function, não posso acessar ela de fora da function
Se eu criar uma variavel fora da function, eu posso acessar ela de dentro de uma 
JavaScript também tem garbage collector, coleta de lixo que funciona mais ou menos assim:
Quando é criada um váriavel local, ela só aparece quando a função é invocada, e se ela não aparecer mais em local nenhum, o JS joga essa variavel fora (elimina da memoria)

importante, sempre use a palavra chave -> var
dá pra criar sem usar a palavra chave var, quando não é usado a palavra chave var, o JS entende que a váriavel esta sendo criada no escopo global

Ou seja, posso criar váriaveis de escopo global dentro de uma function, que supostamente só deveria criar variaveis de escolo local xD

Problema de usar váriaveis globais -> o Js não consegue usar o Garbage Collector em váriaveis de escopo global, ai fode o role, enche a mémoria se a variavel tiver apontada para um objeto grande, fora o conflito de nomes, ter que controlar apontamento
É bom também para identificar a criação de variavel no olhometro, aquela famosa olhadinha no código, é bem nice
